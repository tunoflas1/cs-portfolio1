// interactive

Knight's shortest path

A small live demo of a graph-search idea I enjoy: how few moves does a knight need to get from one square to another? Pick a start, then a target — the board runs a breadth-first search and draws the optimal route. It mirrors the Python version in my repository.

status  click a square to place the knight

BFS guarantees the result is the minimum number of moves, because it explores the board one "ring" of reachable squares at a time. Every square is visited at most once, so it stays fast even across the whole 8×8 grid.
